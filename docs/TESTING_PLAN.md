## 🎯 測試目標

### 主要目標

1. **驗證基本相容性**
   - Francium 能否正常載入真實的 Fabric/Forge 模組
   - 模組載入後遊戲能否正常啟動

2. **測試 AI Bridge 功能**
   - 跨版本模組能否正常執行
   - AI Bridge 的匹配準確率如何
   - 自動生成的適配器是否正確

3. **測試並行加載性能**
   - 並行加載是否真的比循序加載快
   - 不同模組數量下的性能表現
   - 記憶體使用情況

4. **測試雙生態相容**
   - Forge 和 Fabric 模組能否同時載入
   - 兩種生態的模組能否正常交互

5. **發現並記錄問題**
   - 哪些模組可以正常執行
   - 哪些模組有問題
   - 問題的具體表現和原因

## 🖥️ 測試環境

### 硬體要求

| 項目 | 最低配置 | 推薦配置 |
|------|---------|---------|
| CPU | 4 核心 | 8 核心以上 |
| 記憶體 | 8 GB | 16 GB 以上 |
| 硬碟 | 10 GB 可用空間 | SSD，20 GB 以上 |
| 網路 | 可選（用於下載模組） | 寬頻連線 |

### 軟體環境

| 項目 | 版本 |
|------|------|
| 作業系統 | Windows 10/11, macOS 12+, Linux |
| Java | 21 或更高 |
| Minecraft | 1.20.4 / 1.21 |
| Francium | v2.4.0 或最新版本 |

### 測試環境設定

1. **乾淨的 Minecraft 實例**
   - 使用全新的 .minecraft 目錄
   - 避免其他模組加載器的干擾

2. **日誌級別**
   - 設定為 DEBUG 或 TRACE
   - 方便除錯和問題定位

3. **備份機制**
   - 每次測試前備份遊戲目錄
   - 測試失敗時可以快速還原

## 🧪 測試用例

### 測試分類

#### 1. 載入測試（Load Tests）

##### TC-L01: 單模組載入
**目的：** 驗證單個模組能否正常載入
**步驟：**
1. 準備乾淨的 Francium 環境
2. 放入一個測試模組
3. 啟動遊戲
4. 檢查日誌
**預期結果：**
- 模組成功載入
- 沒有錯誤或異常
- 遊戲正常進入主選單

##### TC-L02: 多模組載入
**目的：** 驗證多個模組能否同時載入
**步驟：**
1. 準備乾淨的 Francium 環境
2. 放入多個測試模組（5-10 個）
3. 啟動遊戲
4. 檢查日誌
**預期結果：**
- 所有模組成功載入
- 沒有依賴衝突
- 遊戲正常進入主選單

##### TC-L03: 並行加載性能
**目的：** 測試並行加載的性能提升
**步驟：**
1. 使用相同的模組集合
2. 分別用並行和循序模式加載
3. 記錄加載時間
4. 對比兩者的差異
**預期結果：**
- 並行加載比循序加載快
- 模組越多，提升越明顯

#### 2. 功能測試（Functional Tests）

##### TC-F01: 基本遊戲功能
**目的：** 驗證載入模組後基本遊戲功能正常
**步驟：**
1. 創建新的世界
2. 測試基本操作（移動、挖掘、放置）
3. 測試物品使用
4. 檢查是否有崩潰
**預期結果：**
- 世界可以正常創建
- 基本操作正常
- 沒有崩潰或嚴重錯誤

##### TC-F02: 模組功能驗證
**目的：** 驗證模組的具體功能是否正常
**步驟：**
1. 進入遊戲
2. 測試模組的核心功能
3. 檢查是否有異常
**預期結果：**
- 模組功能正常工作
- 沒有明顯的 bug

#### 3. AI Bridge 測試（AI Bridge Tests）

##### TC-A01: 跨版本載入
**目的：** 測試 AI Bridge 能否讓舊版本模組在新版本上執行
**步驟：**
1. 準備 1.20.4 版本的模組
2. 在 1.21 環境中載入
3. 觀察 AI Bridge 的輸出
4. 檢查是否成功載入
**預期結果：**
- AI Bridge 自動檢測版本差異
- 生成適配器
- 模組成功載入（可能有警告）

##### TC-A02: 方法匹配準確率
**目的：** 評估 AI Bridge 方法匹配的準確率
**步驟：**
1. 選擇一組已知的方法
2. 用 AI Bridge 進行匹配
3. 對比預期結果
4. 計算準確率
**預期結果：**
- 常用方法匹配率 > 90%
- 冷門方法匹配率 > 70%
- 有明確的置信度分數

#### 4. 雙生態測試（Dual Ecosystem Tests）

##### TC-D01: Forge + Fabric 混合載入
**目的：** 測試 Forge 和 Fabric 模組能否同時載入
**步驟：**
1. 同時放入 Forge 和 Fabric 模組
2. 啟動遊戲
3. 檢查兩種模組是否都載入成功
**預期結果：**
- 兩種模組都能載入
- 沒有衝突或崩潰

#### 5. 伺服器同步測試（Server Sync Tests）

##### TC-S01: 基本同步功能
**目的：** 測試伺服器同步功能是否正常
**步驟：**
1. 設定伺服器端的模組清單
2. 用用戶端連接
3. 檢查同步結果
**預期結果：**
- 用戶端自動偵測伺服器模組清單
- 提示需要安裝/更新/移除的模組

## 📝 測試流程

### 測試前準備

1. **環境準備**
   - [ ] 確認 Java 版本正確
   - [ ] 確認 Francium 版本正確
   - [ ] 準備乾淨的測試目錄
   - [ ] 下載測試用的模組

2. **基準測試**
   - [ ] 沒有模組時遊戲能否正常啟動
   - [ ] 記錄基準加載時間
   - [ ] 記錄基準記憶體使用

### 測試執行流程

#### 第一階段：單模組測試

1. 從最簡單的模組開始
2. 逐個測試每個模組
3. 記錄每個模組的測試結果
4. 標記有問題的模組

#### 第二階段：多模組組合測試

1. 組合多個已驗證的模組
2. 測試它們能否一起工作
3. 檢查模組之間的交互
4. 記錄性能數據

#### 第三階段：AI Bridge 測試

1. 測試跨版本模組載入
2. 評估匹配準確率
3. 記錄適配器生成情況
4. 收集失敗案例

#### 第四階段：壓力測試

1. 增加模組數量
2. 測試邊界情況
3. 記錄性能指標
4. 尋找瓶頸和問題

### 測試後整理

1. **結果整理**
   - 彙總所有測試結果
   - 分類問題（嚴重/一般/輕微）
   - 計算成功率

2. **問題分析**
   - 分析失敗的原因
   - 歸類問題類型
   - 提出改進建議

3. **報告生成**
   - 生成測試報告
   - 附上日誌和截圖
   - 提出後續行動建議

## ⚠️ 已知問題

### 已知相容性問題

#### 1. 渲染類模組
**狀態：** ⚠️ 部分相容
**說明：** 深度修改渲染引擎的模組（如 Sodium、Iris）可能會有問題。
**原因：** 渲染相關的方法比較複雜，AI Bridge 的匹配難度較高。
**建議：** 優先測試簡單的模組，再逐步嘗試複雜的。

#### 2. Mixin 複雜度
**狀態：** ⚠️ 部分相容
**說明：** 使用大量複雜 Mixin 的模組可能會有問題。
**原因：** Mixin 的處理比較複雜，某些進階功能可能不支援。
**建議：** 遇到問題可以嘗試調整 Mixin 配置。

#### 3. Forge 模組相容性
**狀態：** 🚧 開發中
**說明：** Forge 模組的支援還在完善中。
**原因：** Forge 的 API 比較複雜，適配層還不完整。
**建議：** 目前優先使用 Fabric 模組，Forge 模組請謹慎測試。

### 性能問題

#### 1. 首次加載較慢
**狀態：** 預期行為
**說明：** 首次加載時 AI Bridge 需要分析和生成適配器，會比較慢。
**建議：** 後續加載會使用快取，速度會快很多。

#### 2. 記憶體使用較高
**狀態：** 預期行為
**說明：** 由於每個模組有獨立的 ClassLoader，記憶體使用會略高一些。
**建議：** 分配足夠的記憶體（建議 4GB 以上）。

## 🔧 除錯技巧

### 啟用除錯日誌

在配置檔案中設定：
```toml
[general]
logLevel = "DEBUG"

[ai-bridge]
debugMode = true
```

### 常用除錯指令

```bash
# 列出已載入的模組
francium list

# 檢查模組依賴
francium deps <mod-id>

# 驗證模組檔案
francium verify <mod-file>

# 分析 AI Bridge 匹配結果
francium bridge analyze <mod-file>
```

### 常見問題定位

1. **模組載入失敗**
   - 檢查日誌中的錯誤訊息
   - 確認依賴是否都已安裝
   - 確認模組版本是否支援

2. **遊戲崩潰**
   - 查看崩潰報告
   - 檢查是否是特定模組導致的
   - 嘗試移除最近新增的模組

3. **AI Bridge 匹配錯誤**
   - 開啟除錯模式
   - 查看匹配日誌
   - 檢查置信度分數
