## ❓ 什麼是混淆名？

Minecraft 的發布版本會進行代碼混淆，將可讀的類名和方法名替換成無意義的名字（如 `m_6057_`、`f_12345_`）。

**舉例：**
- Mojang 名（可讀）：`getDescriptionId`
- 混淆名（發布版）：`m_6057_`

不同版本的 Minecraft，混淆名通常是不同的。

## 📊 目前的狀態

### 1.20.4 映射
- ✅ 方法名和描述符：準確
- ⚠️ 混淆名（obf）：部分準確，部分可能有誤

### 1.21 映射
- ✅ 方法名和描述符：準確
- ⚠️ 混淆名（obf）：目前使用了與 1.20.4 相同的值，**可能不準確**

### 跨版本映射
- ✅ 方法名和描述符對應：準確
- ⚠️ 混淆名對應：取決於兩個版本的 obf 欄位準確性

---

## 🤔 為什麼混淆名可能不準確？

1. **不同版本混淆名不同**
   - Minecraft 每個版本都會重新混淆
   - 1.20.4 和 1.21 的混淆名幾乎肯定不同

2. **手動維護容易出錯**
   - 目前的映射是手動維護的
   - 混淆名數量眾多，容易出錯

3. **缺少自動化工具**
   - 還沒有自動從官方映射生成的工具
   - 需要手動查證每個方法的混淆名

## ⚖️ 混淆名的重要性

### 混淆名在哪些情況下重要？

1. **在混淆後的 Minecraft 中執行時**
   - 如果你使用的是原版 Minecraft（混淆後的）
   - AI Bridge 需要用混淆名來查找方法
   - 這時候混淆名的準確性很重要

2. **開發環境中（使用 Mojang 映射）**
   - 如果你在開發環境中使用 Mojang 映射
   - 方法名是可讀的，不需要混淆名
   - 這時候混淆名的準確性影響不大

### 對 AI Bridge 的影響

| 場景 | 混淆名重要性 | 備註 |
|------|-------------|------|
| 原版 Minecraft 執行 | ⭐⭐⭐⭐⭐ | 必須準確才能找到方法 |
| 開發環境除錯 | ⭐⭐ | 影響較小 |
| 方法相似度計算 | ⭐ | 主要用方法名和描述符 |
| 跨版本匹配 | ⭐⭐⭐ | 兩邊都準確才有用 |

### 目前的影響

**好消息：** 即使混淆名不完全準確，Francium 的很多功能仍然可以正常工作：

- ✅ DAG 並行加載：不受影響
- ✅ SAT 依賴解析：不受影響
- ✅ 套件管理器：不受影響
- ✅ 伺服器同步：不受影響
- ⚠️ AI Bridge（混淆環境）：可能受影響
- ✅ AI Bridge（開發環境）：影響較小

## 🔧 如何獲取準確的混淆名

### 方法一：使用官方映射檔案

Mojang 會發布官方的映射檔案，可以從中提取準確的混淆名。

#### 步驟：

1. **下載官方映射**
   - 從 Minecraft Launcher 或 Maven 倉庫下載
   - 檔案通常是 `.txt` 格式

2. **解析映射檔案**
   - 官方映射格式通常是：
     ```
     com/mojang/blah3rd/Level -> net/minecraft/world/level/Level;
         m_46485_ -> getBlockState
         m_46486_ -> setBlock
     ```

3. **提取混淆名**
   - 找到對應的類和方法
   - 提取混淆名（如 `m_46485_`）

### 方法二：使用 Fabric Yarn 映射

Fabric 社區維護的 Yarn 映射也包含混淆名資訊。

#### 步驟：

1. **下載 Yarn 映射**
   - 從 [FabricMC Yarn](https://github.com/FabricMC/yarn) 下載
   - 或從 Maven: `net.fabricmc:yarn:<version>+build.<build>`

2. **解析 tiny 檔案**
   - Yarn 使用 `.tiny` 格式
   - 包含 official（混淆）、intermediary、named 三層映射

3. **提取混淆名**
   - official 欄位就是混淆名
   - named 欄位是可讀的方法名

### 方法三：使用 Forge MCP 映射

Forge 社區維護的 MCP 映射也可以使用。

#### 步驟：

1. **下載 MCP 映射**
   - 從 [Forge MCPConfig](https://github.com/MinecraftForge/MCPConfig) 下載
   - 或從 Maven 下載

2. **解析 tsrg/srg 檔案**
   - MCP 使用 `.tsrg` 或 `.srg` 格式

3. **提取混淆名**

### 方法四：執行時動態獲取

如果你有運行中的 Minecraft，可以用反射動態獲取混淆名。

```java
// 範例：用反射獲取混淆名
Class<?> blockClass = Class.forName("net.minecraft.world.level.block.Block");
for (Method method : blockClass.getDeclaredMethods()) {
    System.out.println(method.getName() + " " + method.getDescriptor());
}
```

## 🤝 如何貢獻準確的混淆名

如果你發現了錯誤的混淆名，或者想添加更準確的混淆名，歡迎貢獻！

### 貢獻步驟

1. **找到錯誤或缺失的混淆名**
   - 對比官方映射
   - 測試確認問題

2. **修改映射檔案**
   - 編輯 `francium-ai-bridge/src/main/resources/mappings/` 下的 JSON 檔案
   - 更新對應方法的 `obf` 欄位

3. **驗證修改**
   - 確保 JSON 格式正確
   - 測試 AI Bridge 是否正常工作

4. **提交 Pull Request**
   - 描述你做了什麼修改
   - 說明你是如何獲得準確混淆名的
   - 附上參考來源

## 📝 已知問題

### 1.21 混淆名

目前 1.21 的映射中，混淆名使用了與 1.20.4 相同的值，這是**不準確**的。

**狀態：** ⚠️ 已知問題
**影響：** 在混淆環境中，AI Bridge 可能無法正確找到方法
**優先級：** 中高
**計劃：** 未來版本中修復

### 部分冷門方法

一些比較冷門的方法，混淆名可能也不準確。

**狀態：** ⚠️ 可能存在
**影響：** 較小，因為冷門方法使用頻率低
**建議：** 發現問題請及時回報

## ❓ 常見問題

### Q: 混淆名不準確會導致什麼問題？

**A:** 在混淆後的 Minecraft 中，AI Bridge 可能無法正確找到對應的方法，導致跨版本橋接失敗。但在開發環境中（使用 Mojang 映射）影響較小。

### Q: 我可以安全使用 Francium 嗎？

**A:** 可以。即使混淆名不完全準確，大部分功能仍然正常。如果你主要在開發環境中使用，或者不使用 AI Bridge 的跨版本功能，幾乎不會受到影響。

### Q: 什麼時候會修復這個問題？

**A:** 這是一個持續改進的過程。我們會逐步完善混淆名的準確性。歡迎社區貢獻幫助加速這個過程。

### Q: 我可以關閉混淆名相關的功能嗎？

**A:** 目前混淆名是 Mapping 數據庫的一部分，但 AI Bridge 主要還是依賴方法名和描述符進行匹配。如果遇到問題，可以嘗試調整相似度閾值。
